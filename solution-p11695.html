P11695 [JRKSJ ExR] 昼寝 题解

发表于2025-02-12|更新于2025-04-17
|阅读量:

前言

忙里偷闲做 ds！

思路

参考了 @Rainbow_qwq 的题解，orz。

题意简述：支持加 / 删区间和查询当前所有被 [l,r][l,r][l,r] 包含的区间的并是否等于 [l,r][l,r][l,r]。

数据范围不小，很难在线维护，考虑离线分治。

设当前分治区间为 [sl,sr][sl,sr][sl,sr]，中点为 midmidmid，在这一层分治内部处理所有被 [sl,sr][sl,sr][sl,sr] 完全包含并跨过 midmidmid 的询问，其余询问递归处理。考虑两类区间对询问的贡献：跨过 midmidmid 或被 [l,mid][l,mid][l,mid] 和 (mid,r](mid,r](mid,r] 完全包含。

跨过 midmidmid 的操作区间

对于这类区间，当前的目标是对每个询问 [ql,qr][ql,qr][ql,qr] 找到所有被 [ql,qr][ql,qr][ql,qr] 包含的区间中，极左的能贡献的位置 tltltl 与极右的能贡献的位置 trtrtr。下面以求解 tltltl 为例，trtrtr 与 tltltl 本质相同：

首先对时间轴扫描线，在区间被删除的时候撤销其贡献。需要用线段树和可删堆 / set 对每个 lll 维护当前还存在的以 lll 为左端点的区间 [l,r][l,r][l,r] 中 rrr 的最小值。假设现在扫描到时间轴的位置为 iii，需要进行以下操作：

  • 若第 iii 个操作加入了当前需要处理的区间 [l,r][l,r][l,r]，在第 lll 个可删堆中插入 rrr，将线段树下标为 lll 的位置修改为这个堆里的 min⁡\minmin。

  • 若第 iii 个操作是当前需要处理的询问 [l,r][l,r][l,r]，则在线段树上二分 [l,mid][l,mid][l,mid] 内第一个值 ≤r\leq r≤r 的下标，赋值为这个询问的 tltltl。

  • 若第 iii 个操作删除了当前需要处理的区间 [l,r][l,r][l,r]，则在第 lll 个可删堆中删除 rrr，将线段树下标为 lll 的位置更新成这个堆里的 min⁡\minmin（如果堆是空的则更新为极大值）。

可见线段树需要支持的功能为单点修改 / 区间 min⁡\minmin / 区间二分第一个 ≤k\leq k≤k 的数（求解 trtrtr 时正好相反，维护的是区间 max⁡\maxmax 以及区间二分最后一个 ≥k\geq k≥k 的数，同时可删堆维护的也变成了最大值）。

不跨过 midmidmid，被 [l,mid][l,mid][l,mid] 或 (mid,r](mid,r](mid,r] 完全包含的操作区间

这次我们更换扫描线的维度，对序列下标扫描线，线段树维护的下标则是时间轴。还是以处理完全在左边的操作区间为例，完全在右边的情况本质相同。

首先将要处理的询问和区间全部按左端点升序排序，从左到右扫描线。考虑使用这些区间依次往右覆盖询问，遇到询问时判断这个询问被覆盖的值是否小于它的 tl−1tl-1tl−1，若是则不合法。假设当前扫描到序列下标为 iii，需要进行的操作为：

  • 遍历所有左端点为 iii 的询问。设询问编号为 ttt，在线段树上将下标为 ttt 的位置修改为极小值，这样就可以撤销所有左端点 <i<i<i 的区间对这个询问的贡献。

  • 遍历所有左端点为 iii 的区间 [i,r][i,r][i,r]。设其在时间轴上出现的时间区间为 [lt,rt][lt,rt][lt,rt]。在线段树上将 [lt,rt][lt,rt][lt,rt] 对 rrr 取 max⁡\maxmax。

  • 不断取出当前线段树上的最小值，设最小值为 xxx，取到最小值的位置为 ididid，循环终止条件为 x≥ix\geq ix≥i（若 x<ix<ix<i，代表 iii 这个下标未被覆盖，则左端点 ≥i+1\geq i+1≥i+1 的区间就无法对其造成贡献了）。若 tlid>itl_{id}>itlid​>i，这个询问就是不合法的。特别地，若没有任何区间覆盖过这个询问，取出的 xxx 会是极小值。特判一下，这种情况合法的条件为 tlid=lidtl_{id}=l_{id}tlid​=lid​。

这一步需要一个支持区间取 max⁡\maxmax，单点修改和全局最小值的线段树（同样的，求解右区间时也正好相反）。

至此，所有情况的贡献都被我们统计完成。这题里扫描线的各种用法感觉很有启发性。
