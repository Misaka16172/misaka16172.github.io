Cfz Round 3 E / 洛谷 P10033 题解

前言

更孬的阅读体验

我场切了，看来这场题目顺序很诈骗，明明 C 和 D 都没写出来…
等等，这道题竟然评绿了？！
好激动，第一次场切绿，所以来写篇题解。

题意

给定一个 1∼n1\sim n1∼n 的排列 ppp。
你需要构造一个长度为 nnn 的序列 aaa，满足：

  • 序列 aaa 中的每个元素均为不大于 nnn 的正整数；
  • 不存在有序整数二元组 (l,r)(l,r)(l,r)，满足 1≤l≤r≤n1 \le l \le r \le n1≤l≤r≤n 且 ∑i=lrai=∑i=lrpi\sum\limits_{i=l}^r a_i=\sum\limits_{i=l}^r p_ii=l∑r​ai​=i=l∑r​pi​；
    或报告无解。

其中， 1∼n1\sim n1∼n 的排列指满足所有不大于 nnn 的正整数恰好出现一次的序列。

数据范围

设 ∑n\sum n∑n 表示单个测试点中 nnn 的和。
对于所有数据，1≤T≤50001 \le T \le 50001≤T≤5000，2≤n≤1062 \le n \le 10^62≤n≤106，∑n≤106\sum n \le 10^6∑n≤106，保证 ppp 是 1∼n1\sim n1∼n 的排列。

思路

我们可以尝试考虑最终答案序列 aaa 的形式。

aaa 和原序列 ppp 一样都由从 111 到 nnn 的整数构成，但不同点在于 aaa 并不是 1∼n1\sim n1∼n 的排列，aaa 中的数字出现多少次都可以。

而题目里要求 ∀l,r∈[1,n]{\forall}l,r \in [1,n]∀l,r∈[1,n]，∑i=lrai≠∑i=lrpi\sum\limits_{i=l}^r a_i≠\sum\limits_{i=l}^r p_ii=l∑r​ai​​=i=l∑r​pi​ ，直觉告诉我让 ppp 中的全部数字都大于 aaa 就能很轻松地满足这个条件了；

但 pip_ipi​ 的取值最大只能为 nnn，且序列 aaa 里必定有一个 nnn，所以一个全部为 nnn 的 ppp 序列已经可以满足除了 l=r=i(ai=n)l=r=i(a_i=n)l=r=i(ai​=n) 的所有情况了，因为不管怎么取 lll 和 rrr， ∑i=lrpi\sum\limits_{i=l}^r p_ii=l∑r​pi​ 都一定是大于 ∑i=lrai\sum\limits_{i=l}^r a_ii=l∑r​ai​ 的。

那么 ai=na_i=nai​=n 的这一位 pip_ipi​ 该怎么改呢？我们顺着刚刚的思路，也给 iii 这一位放一个除了 nnn 以外尽量大的数，设这个数 pi=n−k(1≤k<n)p_i=n-k (1\leq k<n )pi​=n−k(1≤k<n) 。

那么在取 l,rl,rl,r 时，如果 l>il>il>i 或 r<ir<ir<i，则 ∑p\sum p∑p 的值仍为 (r−l+1)×n(r-l+1)\times n(r−l+1)×n 不受影响，一定满足要求；

但如果 l≤i≤rl\leq i\leq rl≤i≤r， ∑p\sum p∑p 就不一定能够大于 ∑a\sum a∑a 了。
于是我们换个思路，尝试让 pip_ipi​ 的值能够取到一个临界值 [lS,rS][l_S,r_S][lS​,rS​]，正好使这时在 [lS,rS][l_S,r_S][lS​,rS​] 内的 ∑p<∑a\sum p<\sum a∑p<∑a，但凡 lll 再比 lSl_SlS​ 小 111 或 rrr 比 rSr_SrS​ 大 111，∑p\sum p∑p 就会超过 ∑a\sum a∑a。

上述的这个过程我们可以通过从大到小枚举 pip_ipi​ （也就是 n−kn-kn−k ），同时枚举 lll 和 rrr 完成，看起来枚举了三个数很吓人，实际上复杂度并不高，因为临界值很快就会被取到，不信的话可以尝试构造一个能够尽量增加枚举次数的序列 aaa，它大概长成这个样子：

...n−2,n,n−1,n−3,......n-2,n,n-1,n-3,......n−2,n,n−1,n−3,...

可以看到，上面的这个序列已经很努力地在卡掉了 pi(ai=n)p_i(a_i=n)pi​(ai​=n) 在 [n−3,n−1][n-3,n-1][n−3,n−1] 中的值，但这已经是它能做到的极限了，不太优雅的证明如下：

当序列 aaa 能卡掉 pi=n−1p_i=n-1pi​=n−1、n−2n-2n−2 时，n−1n-1n−1 与 n−2n-2n−2 一定正好在 nnn 的两侧，此时 n−3n-3n−3 自然也被卡掉了；

想卡掉 n−4n-4n−4 就只能在 n−1n-1n−1 的同一侧放一个 n−3n-3n−3，但这样做你就永远卡不掉 n−5n-5n−5 了，因为 n−3n-3n−3 与 n−1n-1n−1 在同侧，与 n−2n-2n−2 在异侧；

反之亦然，如果想卡掉 n−5n-5n−5 那 n−4n-4n−4 就一定没法被卡掉。

所以我们不太优雅地证明了在 [n−5,n−1][n-5,n-1][n−5,n−1] 里一定能取到 pip_ipi​。

到这里我们几乎已经解决了本题，接下来考虑序列 aaa 的长度 nnn 太小，导致 n−5<1n-5<1n−5<1 的情况，下面是一个例子：

显然我们取遍了 pip_ipi​ 也没能取到一个值不会被特定的 [l,r][l,r][l,r] 给卡掉，但有了前面的经验，我们只要反过来构造就好了：

把除了 ai=1a_i=1ai​=1 对应的 pip_ipi​ 以外的 ppp 值都设成 111，让 ∑p\sum p∑p 尽量小于 ∑a\sum a∑a，然后再枚举 pip_ipi​ 即可。

用上面的方法就能构造出序列 1 1 1 2，满足条件。

如果上面两种方法都构造不出来序列 ppp，那就是因为序列长度 nnn 过短（也就是 n=2n=2n=2 ），可以证明这个时候一定无解，输出 -1 即可。
